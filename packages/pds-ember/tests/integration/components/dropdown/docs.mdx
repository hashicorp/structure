# Dropdown
`Pds::Dropdown` renders a non-modal dropdown dialog and a trigger for toggling
its visibility.

- [Usage](#usage)
- [Styling](#styling)
- [Markup](#markup)
- [Accessibility](#accessibility)
- [See Also](#see-also)


## Usage

```handlebars
<Pds::Dropdown as |D|>
  <D.Trigger>
    ...
  </D.Trigger>

  <D.Dialog>
    <section>
      <!-- optional -->
      <header>...</header>

      <!-- any of the following -->
      <D.ListItem />
      <LinkTo />
      <Pds::Button />
      <Pds::CtaLink />
    </section>
  </D.Dialog>
</Pds::Dropdown>
```
* `D.isOpen` (boolean) reflects the open state of the Dropdown.
* `D.close` (action) can be used to programmatically close the Dropdown.
* `<D.ListItem>` is a generic wrapper for text content.
* `<D.Trigger>` is a wrapper for a semantic `<summary>` element.
* `<D.Dialog>` is a wrapper for content to be displayed when the dialog is open.
    * Dialog content will vary depending on the use case.
    * Dialog content should be nested in one or more `<section>` elements to
      ensure proper layout and styling.

## Styling
```scss
@use "pds/components/dropdown";
```

### Trigger
Default Button styles are applied to the Trigger.

* It should also support all Button modifier classess.
  (e.g., `.pds--primary`, `.pds--compact`, etc.).


## Markup
The `Pds::Dropdown` component and its subcomponents handle rendering of semantic
markup for a non-modal, dropdown dialog.


## Accessibility

- **WARNING:** not keyboard accessible
    - currently missing `ESC` to close behavior and other keyboard shortcuts
