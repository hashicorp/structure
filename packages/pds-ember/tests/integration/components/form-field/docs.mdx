## Usage
No Ember assets are available.

Continue reading to learn how to apply semantic markup and styling.


## Styling
```scss
@use "pds/components/form-field";
```


## Markup

1. Apply the `.pds-formField` class to a `<div>` around field children.
    - Normal document flow cannot be assumed, so a DIV is required in order to
      isolate any apply any advanced layout strategies.

Children should appear in the following order:

1. [Field Name](?path=/docs/components-field-name--index)
2. [Help Text](?path=/docs/components-help-text--index)
3. Control
4. [Error Message](?path=/docs/components-error-message--index)

For the majority of use cases, this can be achieved by defining markup in
explicit DOM order.  Children of `.pds-formField` are arranged vertically,
in DOM order.

### Input Field

* Control: [Input](?path=/docs/components-input--index)

```html
<div class="pds-formField">
  <label for="{INPUT_ID}" class="pds-fieldName">
    ...
  </label>

  <Pds::HelpText id="{HELP_ID}">
    ...
  </Pds::HelpText>

  <Pds::Input
    @type="text"
    aria-describedby="{HELP_ID} {ERROR_ID}"
    id="{INPUT_ID}"
  />

  <Pds::ErrorMessage id="{ERROR_ID}">
    ...
  </Pds::ErrorMessage>
</div>
```

## Accessibility

1. Make sure to explicitly associate the control with a `<label>` via
   `<control-element id="{ID}" />` and `<label for="{ID}">`.
2. Make use of `aria-describedby` to associate contextual descriptions with the
   generated control element.
