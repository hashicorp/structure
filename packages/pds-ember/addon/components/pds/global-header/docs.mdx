## Usage

```handlebars
<Pds::GlobalHeader>
  <:logo>
    <img
      src="@hashicorp/pds-ember/logomarks/HashiCorp_White.svg" alt="logo"
    />
  </:logo>

  <:primary-nav>
    <LinkTo @route="...">...</LinkTo>
    ...
  </:primary-nav>

  <:secondary-nav>
    <LinkTo @route="...">...</LinkTo>
    ...
  </:secondary-nav>

  <:switcher>
    <Pds::Dropdown />
  <:switcher>

  <:user-menu>...</:user-menu>

  <:extra>
  {{!-- extra content --}}
  </extra>
  ...
</Pds::GlobalHeader>
```

## Styling
```scss
@use "pds/components/global-header";
```
* Extra content will be slotted into a single area of the grid layout, so it's
  recommended that you wrap extra content in a `<div>` to avoid unexpected
  grid placement.


## Markup

GlobalHeader uses several named blocks to slot content into the appropriate
CSS Grid areas. The layout is as follows:

```
+---------+-------------+----------------+----------+------------------+--------------+
| <:logo> | <:switcher> | <:primary-nav> | <:extra> | <:secondary-nav> | <:user-menu> |
+---------+-------------+----------------+----------+------------------+--------------+
```

All blocks are optional and when omitted, will not render any wrapping elements.

* `<:logo>` yields into a `<div>` element
* `<:switcher>` yields to a `<div>` element suitable for placing a
    [Dropdown](?path=/docs/components-dropdown--index)
    - It's primarily used as a means to switch application contexts.
* `<:primary-nav>` yields content to a semantic `<nav>` element
    - It's primarily used to navigate to different contexts within an application.
* `<:secondary-nav>` yields into a semantic `<nav>` element
    - It's primarily used to provide helpful links that aren't specific to a
      particular application context (e.g., "support", "notifications", etc.)
* `<:user-menu>` is suitable for rendering a [Dropdown](?path=/docs/components-dropdown--index)
  component.
    - It's primarily used to provide information and actions related to the
      logged in user account.
