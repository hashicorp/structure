# Nav
Wrapper for semantic `<nav>` element with scoped styling applied.

- [Styling](#styling)
- [Markup](#markup)
- [Accessibility](#accessibility)
- [See Also](#see-also)


## Usage

```handlebars
<Pds::Nav>
  <section>
    {{! optional header }}
    <header>Recommended Markup</header>

    {{! LinkTo is recommended (automatically applies "current" styling) }}
    <LinkTo @route="...">Link 1</LinkTo>
    <LinkTo @route="...">Link 2</LinkTo>
    ...
    <LinkTo @route="...">Link N</LinkTo>
  </section>

  <section>
    <header>Alternative Markup</header>
    ...

    <Pds::ExternalLink href="...">
      Link to external URL
    </Pds::ExternalLink>

    {{! doesn't apply "current" appearance without custom logic }}
    <a href="...">
      manually generated hyperlink
    </a>
  </section>

  ...
</Pds::Nav>
```

## Styling
```scss
@use "pds/components/nav";
```
**NOTE:** While it is possible to use this component for horizontally arranged
navigation (i.e., nav bar), it currently only applies a vertical layout,
because its primary use is as sidebar navigation.

Styling is optimized for use with `<LinkTo>` hyperlinks so that "current"
appearance is applied via the `.active` class.


## Markup
The `<Pds::Nav>` component handles generation of semantic HTML `<nav>` markup.

* All links must be nested within `<section>` children.
* Section titles (`section > header`) are optional, but recommended.
* `<LinkTo>` is recommended for generating hyperlinks, but links may be
  generated however you see fit.
