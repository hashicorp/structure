## Usage
No Ember assets are available.

Continue reading to learn how to apply semantic markup and styling.


## Markup
Use a semantic heading element.

```html
<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>
```


## Styling
```scss
@use "pds/components/heading";
```

Styles for all semantic heading elements are reset when you `@use` provided
styles.

Reset styles include:

* font
* line-height
* margin
* padding


As such, you'll need to apply the `pds-heading` base class along with a modifier
class in order to apply a specific heading style.

- `.pds-heading.pds--0` (Dashboard Title)
- `.pds-heading.pds--1` (Page Title)
- `.pds-heading.pds--2` (Section Title)
- `.pds-heading.pds--3` (Sub-section Title)
- `.pds-heading.pds--4` (Sub-sub-section Title)


## Accessibility
* Make sure your semantic headings (`<h1>` through `<h6>`) are defined in
  increasing order without skipping a level.
    * e.g., don't use an `<h2>` followed by an `<h4>`
